## 🧭 탐색 알고리즘 로직 (Search Algorithm Logic)

이 프로젝트는 **Ver3: Greedy + Backtracking** 알고리즘을 사용하여 실종자(Wi-Fi 핫스팟 신호원)를 추적합니다. 사용자는 앱의 지시에 따라 이동하며 RSSI(신호 강도)를 수동으로 입력해야 합니다.

### 🔄 작동 프로세스 (Process Flow)

1.  **초기화 (Initialization)**
    * GPS 신호가 안정화될 때까지(샘플 20개 수집) 대기합니다.
    * 시작 위치에서 RSSI를 입력합니다. (단, -80dBm 미만의 약한 신호에서는 시작되지 않습니다.)

2.  **탐색 단계 (Scanning Phase)**
    * **Outbound:** 현재 중심(Center)에서 **북(N)쪽 20m** 지점으로 이동을 지시합니다.
    * **Scan:** 목표 지점에 도착하면 사용자가 현재 위치의 **RSSI 값**을 입력합니다.
    * **Inbound:** 다시 중심(Center)으로 복귀를 지시합니다.
    * **Repeat:** 위 과정을 **남(S) → 동(E) → 서(W)** 순서로 반복하여 4방향의 신호를 수집합니다.

3.  **판단 및 이동 (Decide & Move)**
    * **Decide:** 4방향 중 가장 신호가 강했던 방향을 선정합니다.
    * **Greedy Move (직진):** 선정된 방향으로 중심(Center)을 이동시킵니다. 만약 이동한 곳에서 신호가 더 강해졌다면, 주변 탐색을 생략하고 **즉시 해당 방향으로 추가 직진**합니다.
    * **Backtrack (후퇴):** 직진했으나 신호가 약해진 경우, 즉시 이전의 좋았던 위치로 복귀하여 다시 탐색을 시작합니다.

4.  **종료 (Termination)**
    * **Loop:** 목표 RSSI 값(**-40dBm 이상**)에 도달할 때까지 위 과정을 반복합니다.
    * 도달 시 "탐색 성공" 알림과 함께 종료됩니다.

## 🛡️ 오차 보정 기술 (Error Correction)

야외 도보 테스트의 정확성을 높이기 위해 다음과 같은 필터링 기술이 적용되었습니다.

* **속도 제한 (Speed Limit):** 이동 속도가 **3.0m/s (약 10.8km/h)**를 초과하면 GPS 튐(Jitter) 현상으로 간주하여 무시합니다.
* **완전 고정 (Hard Lock):** 사용자가 제자리에 멈추면(가속도 센서 기반), GPS 좌표가 흔들리지 않도록 지도상의 위치를 강제로 고정합니다.
* **칼만 필터 (Kalman Filter):** 이동 중 발생하는 GPS 노이즈를 제거하여 경로를 부드럽게 보정합니다.
